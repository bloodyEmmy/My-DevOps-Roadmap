#flashcards/Docker
***
Отдельное место для ***постоянного*** хранения данных, управляемое [[Docker]] - каталог в той части хост-системы, где находится Docker. 
Он подключен к каталогу в контейнере, чтобы можно было записывать и считывать данные.
В этих местах данные продолжают жить после удаления [[Контейнер|контейнера]] + используются для монтирования.
***
***Проще.***
Просто папка, созданная [[Dockerfile]] при помощи [[VOLUME команда Dockerfile|VOLUME]] или нами при запуске контейнера.
- Используется для записи больших объемов данных.
- Можно смонтировать один том для нескольких контейнеров одновременно => контейнеры могут обмениваться данными между собой.
***
Для записи больших данных используются механизмы [[Bind Mount (Монтирование каталогов с хоста)|бинда]] и [[Tmpfs (Temporary file system)|tmpfs]], которые отличаются от Тома.
***
Отличается от Бинда тем, что Docker автоматически создает том и подключает его к хосту, хранит инфу про каждый том в JSON ([[Inspect команда Docker|inspect]]), может хранить тома не только на локальном диске, но и облаке, сам управляет жизненным циклом тома и др.
- Адрес тома - `/var/lib/docker/volumes`. Именно этим и отличается от бинда и хранении в [[Оперативная память (ОЗУ)|ОЗУ]] - туда никто не может попасть, кроме [[Суперпользователь (root)|суперпользователя]] => высокая изоляция и кроссплатформенность + нет нагрузки на [[Файловая система|файловую систему]] лишними поисками.
***
***Пример подключения тома.***
```bash
sudo docker run -dit --name mynginx --mount source=myvol2,target=/test nginx:latest
```
- `source=myvol2` - имя тома.
- `target=/test` - путь внутри контейнера.
***
***Адрес томов (через Snap).***
`/var/snap/docker/common/var-lib-docker/volumes`
