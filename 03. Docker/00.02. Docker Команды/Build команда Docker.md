#flashcards/Docker
***
- Собирает новый [[Образ контейнера (Docker Image)|образ]] из [[Dockerfile]] и контекста сборки путем чтения Dockerfile последовательно построчно.
- Создает последовательность [[Слой (layer)|слоев]] образа на основе инструкций [[FROM команда Dockerfile|FROM]], [[RUN команда Dockerfile|RUN]], `COPY` и др.
- Использует [[Кэш в Docker|кэш]], если слои не изменились.
***
***Синтаксис.***
```bash
docker build <флаги> <путь>
```
`<путь>` - директория, из которой [[Docker]] возьмет:
- Dockerfile (по умолчанию `.`).
- Все файлы, которые упоминаются в [[ADD команда Dockerfile|ADD]] / [[COPY команда Dockerfile|COPY]].
***
***Флаги.***
1. `-t (--tag)` - присваивает имя и тег образу.
2. `--no-cache` - игнорировать кэш и пересобрать слои заново.
3. `-f (--file)` - указать путь к Dockerfile, если он не в `.`.
