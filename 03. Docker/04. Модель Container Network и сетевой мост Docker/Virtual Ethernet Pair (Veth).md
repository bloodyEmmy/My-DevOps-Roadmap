***
***Расшифровка.*** Virtual Ethernet Pair
- Виртуальный сетевой кабель в Linux, который идет парой `vethA  <-->  vethB`. Пакеты, отправленные на один конец, мгновенно появляются на другом.
- Используется в соединении между изолированным [[Контейнер|контейнером]] и основной сетью хоста через [[Сетевой мост (Bridge) Docker|сетевой мост]], а также для разделения сети через [[Пространства имен (namespaces)|пространства имен]].
- Каждый контейнер получает свою veth-пару: `eth0` внутри контейнера и `vethX` на хосте, подключенному к мосту.
***
***Внутри контейнера (eth0).***
1. Один конец виртуального кабеля помещается внутрь сетевого пространства контейнера.
2. [[Docker]] переименовывает его в `eth0`.
3. Ему назначается внутренний [[IP-адрес]] (прим: `172.17.0.2`).
4. Для [[Процесс|процессов]] внутри контейнера он выглядит как обычная физическая [[Сетевая карта|сетевая карта]].
***
***На хост-машине (vethX).***
1. Второй конец остается в глобальном сетевом пространстве.
2. Получает случайное имя (прим: `veth1234abc`).
3. Автоматически подключается к виртуальному [[Коммутатор (switch)|коммутатору]] [[docker0]].
