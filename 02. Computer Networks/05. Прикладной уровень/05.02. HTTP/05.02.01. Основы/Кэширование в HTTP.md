#flashcards/network 
***
Механизм, который позволяет браузеры (или промежуточным [[Сервер (Клиент Серверная Архитектура)|серверам]]) сохранять копии ответов от Web-сервера, чтобы не скачивать их заново при повторных запросах.
- В [[Протокол уровня|протоколе]] [[Протокол HTTP|HTTP]] [[Кэширование|кэширование]] управляется с помощью [[Заголовок|заголовков]] в HTTP-пакетах.
- Работает точно так же, как и [[Domain Name System (DNS)|DNS]]-кэширование, которое используется в [[Рекурсивный DNS сервер|рекурсивных серверах]].
***
[[Клиент (Клиент Серверная Архитектура)|Клиент]] как-то должен понимать, нужно ли брать данные из кэша или они поменялись и надо идти на сайт.
- Для этого используется [[Методы (запросы) HTTP|запрос]] `GET` с условием - запрос Web-серверу передать ресурс, если он изменился с указанного времени.
- Если ресурс не изменился, то посылается [[Статусы HTTP|статус]] ответа `304`.
- Если изменился, то статус ответа = `200`.
