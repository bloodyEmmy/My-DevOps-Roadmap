#flashcards/network 
***
Стандартный тип подключения в [[Протокол HTTP|HTTP]]/1.1.
***Алгоритм работы.***
1. Браузер открывает одно [[Протокол TCP|TCP]]-соединение и не [[Установка и завершение TCP соединения (рукопожатия)|закрывает]] его после получения первого файла (в отличие от [[Непрерывные (короткие) HTTP соединения|короткого HTTP соединения]]). Через этот канал последовательно скачиваются все элементы Web-страницы.
2. В пакете HTTP за это отвечает [[Заголовок|заголовок]] `Connection: keep-alive`. Соединение живет до тайм-аута (бездействие), после чего [[Сервер (Клиент Серверная Архитектура)|сервер]] его закрывает.
3. Как итог - нет необходимости начинать передачу с малым размером [[Скользящее окно|окна]] TCP (то есть [[Медленный старт (Метод TCP)|медленный старт]]).
4. Минус - запросы идут по очереди => большой файл может застрять и тогда все остальные файлы будет ждать своей очереди.
