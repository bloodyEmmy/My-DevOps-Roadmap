#flashcards/network 
***
[[Прокси сервер (Proxy)|Прокси]], который работает на стороне [[Клиент (Клиент Серверная Архитектура)|клиента]].
- Клиент сам настраивает его в [[Операционная система (ОС)|ОС]] или браузере.
***
***Роли.***
1. ***Скрытие [[IP-адрес|IP-адреса]]*** - когда клиент шлет запрос через прокси, финальный сайт видит IP самого прокси-[[Сервер (Клиент Серверная Архитектура)|сервера]], а не клиента. IP клиента (выданный провайдером по [[Протокол DHCP|DHCP]]) остается в секрете.
2. ***Обход блокировок*** - если сайт заблокирован на территории страны провайдером через [[Domain Name System (DNS)|DNS]], то можно взять прокси в другой стране.
3. ***Корпоративный прокси*** - в крупных компаниях админ настраивает прокси так, чтобы тот вскрывал [[Протокол HTTP|HTTP]]-пакеты, читал [[Uniform Resource Locator (URL)|URL]] и блокировал [[Методы (запросы) HTTP|запросы]] к соцсетям или играм во время рабочего дня.
4. ***Глобальное [[Кэширование в HTTP|HTTP-кэширование]]*** - если много сотрудников скачивают один файл, то прокси скачивает его 1 раз и сохраняет в память для раздачи остальным пользователям.
