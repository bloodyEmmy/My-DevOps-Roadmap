#flashcards/databases 
***
Выделенный сектор в [[Оперативная память (ОЗУ)|ОЗУ]] [[Сервер (Клиент Серверная Архитектура)|сервера]], к которому имеют одновременный доступ все [[Процесс|процессы]] внутри базы данных.
- Важнейший компонент для производительности [[СУБД (Системы Управления Базами Данных)|СУБД]].
- Используется по той причине, что каждое новое подключение - отдельный изолированный процесс [[Операционная система (ОС)|ОС]] и им нужно через что-то общаться без дублирования информации в ОЗУ.
***
***Устройство.***
1. ***Буферный [[Кэширование|кэш]]*** - самый большой кусок разделяемой памяти для страниц данных. Когда процесс читает строку из таблицы, он сначала лезет в кэш, после чего при ее отсутствии, читает с диска и копирует в кэш.
2. ***Буфер [[WAL (Write Ahead Logging)|WAL]]*** - временная память для логов.
3. ***Массив данных*** о состоянии всех [[Транзакция|транзакций]].
4. ***Пространство блокировок*** - если один процесс работает с данными, то он блокирует их, чтобы другие процессы не работали с ними параллельно => решение [[Состояние гонки (Race Condition)|состояния гонки]].
